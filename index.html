Мой опыт
с AI

Что использую, что работает, что нет

Что я использую каждый день

Raycast

Быстрые запросы к AI прямо с клавиатуры. Простой чат, перевод, генерация — без переключения контекста

Claude Code

Основной рабочий инструмент. Пишет, рефакторит, деплоит — как второй разработчик в терминале

Granola

Транскрибация встреч. Записывает, расшифровывает, делает саммари — автоматически

VS Code

IDE для кода. Почти не использую — Claude Code закрывает 90% задач прямо в терминале

Что не работает

  • Пытаться использовать 10 инструментов сразу — хватит 2-3
  • Ждать идеального результата с первого промпта
  • Использовать AI без контекста — чем больше он знает, тем лучше
  • Бояться терминала — это просто текстовый интерфейс

Главный инсайт: AI снимает вопрос «как сделать?» — но не снимает вопрос «а нужно ли это делать?». Думать головой придётся всё равно. И это хорошо.

AI убирает сложность реализации. Остаётся смысл

AI как второй мозг

Два подхода — оба работают. Главное — дать AI контекст.

Notion AI

Подход Саши (CEO)

  • Все документы, созвоны, задачи — в одном месте
  • AI видит весь workspace и может связывать данные
  • Транскрибация встреч + авто-запись через Notion Calendar
  • Спрашиваешь: «что обсуждали на созвоне?» — AI знает

Claude Code + Память

Мой подход (CTO)

  • Каждый проект — папка, Claude знает её структуру и стек
  • Файловая память: кто я, мои предпочтения, реестр всех проектов
  • 30+ проектов в реестре — Claude помнит стек, статус, трекер каждого
  • Telegram-канал через --channels — пишу задачи из чата

Как устроена моя система

Структура памяти

~/.claude/projects/
  project/
    memory/
      MEMORY.md  # индекс
      projects/
        registry.md # все 30+ проектов
  project-agenthub/  # контекст
  project-zhk-saas/  # контекст
  project-bicycle-training/
  ... ещё 30+ проектов

Что Claude знает обо мне

  • Роль, стек, GitHub аккаунты
  • Какой трекер у какого проекта (Huly / GitHub Projects)
  • Статус каждого проекта: active, support, done, frozen
  • Мои предпочтения: как коммитить, как называть ветки, как писать PR

Чем дольше работаешь — тем умнее становится. Claude учится на каждой сессии.

CLAUDE.md — инструкция для AI

Один файл в корне проекта — и Claude знает всё, что нужно для работы с ним.

# CLAUDE.md

## Стек
Nuxt 4, Hono, Drizzle, oRPC, Turbo

## Команды
dev: pnpm dev
build: pnpm build
lint: pnpm lint

## Правила
- Используй composables из /shared
- API через oRPC, не fetch
- Коммиты на русском

Что даёт

  • Claude сразу знает как запустить проект
  • Пишет код в вашем стиле, не угадывает
  • Не нужно каждый раз объяснять контекст
  • Работает для всей команды — файл в репозитории

У каждого моего проекта есть CLAUDE.md. Claude сам предлагает обновлять его, когда узнаёт что-то новое.

MCP — подключай что угодно

MCP (Model Context Protocol) — открытый стандарт. Claude подключается к внешним сервисам как к розеткам: один протокол, сотни интеграций.

Что подключено у меня

☷
Huly — трекер задач, issues, статусы
⚙
Chrome DevTools — дебаг, скриншоты, консоль
◆
Figma — дизайн-макеты в код
✉
Telegram — канал для задач из чата
☀
Google Calendar — события, расписание

Как это работает

# Claude видит инструменты MCP-серверов
# как свои собственные способности:

mcp__huly__create_issue(...)
mcp__chrome__take_screenshot()
mcp__figma__get_design(...)
mcp__telegram__send_message(...)

Суть: MCP — открытый стандарт. Любой сервис может создать коннектор. Уже есть сотни: Slack, Jira, Notion, базы данных, API — что угодно. Подключил — Claude умеет с ним работать.

Что такое терминал

Терминал — текстовый способ управлять компьютером.

Вместо кликов по папкам — пишете команды. Как адресная строка в проводнике, но умеет гораздо больше.

# перейти в папку
~$ cd Projects

# посмотреть файлы
~/Projects$ ls
my-site/ dashboard/ bot/

# запустить Claude Code
~/Projects$ claude

Какой терминал использовать

 macOS

Ghostty / Warp / Terminal

  • Ghostty — быстрый, минималистичный
  • Warp — красивый, с AI-подсказками
  • Terminal.app — уже встроен
☰ Windows

Windows Terminal + PowerShell

  • Windows Terminal — скачать из Microsoft Store
  • PowerShell — уже встроен
  • Обязательно: Git for Windows

~$ curl -fsSL https://claude.ai/install.sh | bash

# или через Homebrew
~$ brew install claude-code

Одна команда. Готово.

☰ Windows

# 2. В PowerShell:
PS> irm https://claude.ai/install.ps1 | iex

# 3. Перезапустить терминал!

Важно: Git обязателен. После установки — перезапустить терминал.

Генератор стека за одну команду

Запустить проект — минута. Понять, что строить — это и есть работа

Better-T-Stack — генератор проектов

Выбираете технологии на сайте — получаете готовый проект с одной команды в терминале.

~$ pnpm create better-t-stack@latest my-app \
  --frontend nuxt --backend hono --runtime bun \
  --api orpc --auth better-auth --database sqlite \
  --orm drizzle --package-manager pnpm --addons turborepo

Конфигуратор: better-t-stack.dev/new

Наш стек — что за что отвечает

Nuxt

Фронтенд-фреймворк. Рисует страницы, роутинг, SSR — то, что видит пользователь

Hono

Бэкенд-сервер. Обрабатывает запросы, API, бизнес-логика — мозг приложения

oRPC

Связь фронта с бэком. Типобезопасные вызовы — фронт знает, что бэк вернёт

Drizzle

ORM — работа с базой данных через TypeScript вместо SQL-запросов

SQLite

База данных. Файл на диске, не нужен отдельный сервер — просто работает

Better Auth

Авторизация. Логин, регистрация, сессии — вся безопасность из коробки

Bun — быстрый рантайм pnpm — менеджер пакетов Turborepo — монорепо, всё в одном

Не только серьёзное — оставляйте время на fun

Каждую неделю AI генерирует коллаж по итогам нашего weekly-митапа. Скрипт подключается к Granola, читает транскрипцию, вытаскивает «прикольное» и рисует картинку через Gemini.

Не всё должно быть продуктивным. Эксперименты — это тоже работа

#weekly_adventures — как это устроено

Пайплайн

● Weekly-митап
Команда делится прикольным за неделю
↓
● Granola записывает и транскрибирует
Автоматически, без кнопок
↓
● Claude Code + скилл /weekly-art
Ищет встречу, читает транскрипт, вытаскивает fun facts
↓
● Gemini рисует коллаж
Скрапбук-стиль, каждый факт — карточка
↓
● Пост в Telegram
Чат Bureau Websites + #weekly_adventures

Зачем?

  • Это весело — команда ждёт пост каждую неделю
  • Это эксперимент — связал 4 сервиса: Granola, Claude, Gemini, Telegram
  • Это полезно — понял как работают MCP, API картинок, пайплайны
  • Заняло пару часов — один вечер и скрипт готов

Не бойтесь делать бесполезные вещи. Именно из экспериментов рождаются самые интересные идеи и навыки.

Главная суть

AI снял барьер сложности. Раньше фича не делалась, потому что слишком сложно.

Теперь вопрос другой: а нужно ли это вообще?

AI не заменяет экспертизу — он усиливает её. Думать головой придётся всё равно. И это самое ценное.
